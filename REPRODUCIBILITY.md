# Reproducibility

Every number and figure in the paper/README is regenerated by the commands below. No data is vendored.

## Environment
- Python ≥ 3.9; `pip install -e .` (numpy, scipy; dev extras: pytest, matplotlib, pandas).
- Synthetic results are deterministic (fixed seeds). Real-data results depend on the downloaded traces.
- Hardware used for the published real-data runs: AWS EC2 `c5.2xlarge` spot (8 vCPU, 16 GB), Ubuntu
  22.04, ap-south-1 (torn down after the run). RocksDB: `db_bench` from `rocksdb-tools` 6.11.4.

## Synthetic (laptop, < 1 min) — H0, H1, H2 + negative controls
```
pip install -e .
pytest -q                      # 26 tests incl. NC1/NC2 and the closed-form checks
python bench/run_synthetic.py  # writes results/{H0,H1,H2}.json, figures/{loglaw,robustness,frontier}.png
```

## Real Twitter cache traces (H2c) — needs disk + bandwidth
```
# stream a prefix of 3 time-sorted production clusters (no full 20GB download)
bench/fetch_data.sh ./data 110000000 45 34 22
python bench/run_traces.py --data-dir ./data --fmt twitter --R 256 --bpk 10 \
       --window 200000 --max-windows 400 --group 1
# -> results/H2c.json, figures/real_frontier.png
```
Traces: Twitter production cache traces, Yang et al., OSDI'20
(https://github.com/twitter/cache-trace, hosted at CMU PDL). A Wikipedia-pageviews fallback
(`--fmt wiki`) is also supported for environments without the Twitter archive.

## Real RocksDB engine (H3)
```
sudo apt-get install -y rocksdb-tools        # provides db_bench
bench/rocksdb_fpr.sh results/rocksdb_fpr.json 3000000 2000000
python bench/plot_rocksdb.py                  # -> results/rocksdb_summary.json, figures/rocksdb_fpr.png
```

## Provenance
- Trace files are content-addressable by cluster id + prefix length; `fetch_data.sh` records line
  counts. RocksDB runs log `db_bench` version and full statistics.
- Seeds: synthetic runners use fixed `numpy.random.default_rng` seeds (see `lsm_bloom/experiment.py`).
